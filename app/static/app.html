Instructions

Golden path — the usual flow

  1. Import — drag a CSV/XLSX onto the Import tab, or paste a shared-drive URL. Validation runs automatically right after.
  2. Check the results table — rows land in Valid, Invalid, Duplicate, Existing Customer, or Already Contacted. Nothing is dropped, everything is visible.
  3. Create or pick a campaign and queue the Valid rows into it — this also auto-generates a draft email per prospect.
  4. Switch to the Campaigns tab to review queued drafts: edit subject/body, then Approve or Reject each one.
  5. Send the campaign — this delivers every Approved draft via Gmail. Requires email to be configured first (see Admin below).
  6. Watch progress on the Dashboard — the outreach funnel, replies, and audit log update live.
  7. When a prospect replies with interest, mark them Quote Requested, then Won or Lost from the Campaigns tab to track real business value.

Important details

  • Sending email requires setup. Add GMAIL_ADDRESS and GMAIL_APP_PASSWORD (a Gmail App Password, not your real password) as environment variables, then restart the server. Until then, everything except the actual send/reply-poll works normally.
  • Suppression is checked twice — once when queuing into a campaign, again right before sending — so an opt-out can never slip through even if it happens after a draft was already approved.
  • Opt-outs are also detected automatically from reply text (e.g. "unsubscribe," "remove me," "do not contact") and added to suppression without any manual step.
  • Existing customers are matched against a seeded customer list and marked instead of queued for outreach.
  • Already-contacted prospects are caught on re-import. Email is the key: if an address was already sent an outreach email in any prior campaign — even from a completely different import file — re-importing it now flags it Already Contacted instead of validating as a fresh lead. Only prospects that have actually received a sent email are caught this way; someone still sitting Queued/Approved (never sent) in another campaign won't be flagged by this check.
  • Duplicates are flagged within the same import batch (same email seen twice in one file).
  • Data persists in a SQLite file across restarts as long as the server always runs with the same APEX_DB_PATH — back it up periodically since there's no separate hosted database.
  • The top-bar badge shows connection state: WAKING SERVER… means it's retrying while a sleeping backend (e.g. Render free tier after inactivity) spins back up — this can take up to ~90 seconds on the first load. DEMO MODE — SERVER UNREACHABLE means every retry failed — everything you do is kept only in this browser tab and is lost on refresh until the server responds; it keeps retrying quietly in the background and switches over automatically once reachable. CONNECTED TO SERVER means everything you do is saved for real.
